# Расчёт дневной волатильности тикера (MOEX, boardid=TQBR) по данным **только основной сессии**

## Цель
Дать простую и устойчивую методику расчёта **дневной волатильности** конкретного тикера на Московской бирже по сделкам **только основной торговой сессии** за указанную дату. На вход подаются дата и тикер (boardid = `TQBR`). Баланс: простота > предельная точность.

---

## Ключевая идея
Использовать **оценку Паркинсона** по High/Low внутри основной сессии. Как альтернативу (лучше при тренде внутри дня) — **оценку Rogers–Satchell (RS)** по `O/H/L/C` основной сессии.

---

## Обозначения
- `O` — цена первой сделки **внутри основной сессии** за дату (включая сделку аукциона открытия, если она относится к основной сессии).
- `H` — максимальная цена сделки внутри основной сессии за дату.
- `L` — минимальная цена сделки внутри основной сессии за дату.
- `C` — цена последней сделки **внутри основной сессии** за дату (включая сделку аукциона закрытия, если она относится к основной сессии).
- `ln` — натуральный логарифм.
- Все цены — в тех единицах, в которых котируется инструмент на TQBR. Валюта не влияет на расчёт, так как используются относительные изменения.

> **Важно:** использовать только сделки из основной сессии по тикеру на `boardid = TQBR`. Утреннюю и вечернюю сессии исключать. Временная зона — `Europe/Moscow` (биржевая). Не хардкодить часы — ориентироваться на биржевые метки сессии в данных за конкретную дату.

---

## Формулы

### 1) Волатильность Паркинсона (дневная)
\[
\sigma_{\text{day}}=\frac{\left|\ln\!\left(\frac{H}{L}\right)\right|}{2\sqrt{\ln 2}}
\]
Числовая константа: \(\frac{1}{2\sqrt{\ln 2}}\approx 0.6005612044\).

**Плюсы:** минимальные требования к данным (`H` и `L`).  
**Минусы:** чувствительна к выбросам в экстремумах.

### 2) Волатильность Rogers–Satchell (дневная)
\[
\sigma_{\text{day}}=\sqrt{\ln\!\frac{H}{C}\cdot\ln\!\frac{H}{O}+\ln\!\frac{L}{C}\cdot\ln\!\frac{L}{O}}
\]

**Плюсы:** устойчивее к внутридневному тренду, использует `O/H/L/C`.  
**Минусы:** нужны все четыре цены основной сессии.

### Аннуализация (при необходимости)
\[
\sigma_{\text{ann}}=\sigma_{\text{day}}\sqrt{252}
\]
Для ежемесячной шкалы можно использовать \(\sqrt{21}\), для недельной — \(\sqrt{5}\).

---

## Шкала результата 0–100 (требование)
Возвращайте **процент** дневной волатильности, жёстко ограниченный диапазоном 0–100.

\[
\text{score}_{0\text{–}100}=\min\!\bigl(100,\;\max(0,\; \sigma_{\text{day}}\cdot 100)\bigr)
\]

- \(\sigma_{\text{day}}\) — оценка Паркинсона или RS за **эту** дату и **эту** основную сессию.  
- Аннуализацию для этой шкалы **не применять**.  
- Округление: по бизнес‑требованию. Примеры — до 2 знаков после запятой или до целого 0–100.

---

## Алгоритм извлечения цен из данных основной сессии
1. Отфильтровать сделки по:
   - `boardid = TQBR`,
   - заданному тикеру,
   - заданной дате,
   - признаку «основная сессия» от биржи. Исключить сделки утренней и вечерней сессий.
2. Сформировать `O/H/L/C` для **этой** сессии и даты:
   - `O` — цена первой сделки основной сессии (включая аукцион открытия, если в метках указано, что он относится к основной сессии).
   - `H` — максимум цены сделки в основной сессии.
   - `L` — минимум цены сделки в основной сессии.
   - `C` — цена последней сделки основной сессии (включая аукцион закрытия, если он относится к основной сессии).
3. Если есть ровно одна сделка в основной сессии, то `H = L`, следовательно \(\sigma_{\text{day}} = 0\).

> **Примечания по качеству данных**
> - Использовать только **фактические сделки**. Котировки из стакана не подходят.
> - Если в потоке есть спец‑режимы, адресные или крупные внебиржевые сделки, которые технически попали в выборку, но не отражают обычную ликвидность, их лучше исключить на уровне режима/типа сделки согласно вашей бизнес‑логике.
> - Для стабильности расчётов округление применять **только на выводе**. Внутри вычислений — `float64`.
> - Проверять, что `H ≥ max(O, C)` и `L ≤ min(O, C)` для самоконтроля данных.

## Пошаговый пример расчёта

Предположим, что по тикеру в основной сессии за дату получены:
- `O = 100.00`
- `H = 104.50`
- `L = 98.70`
- `C = 103.20`

### Паркинсон
1. Лог‑размах: \(\ln(H/L)=\ln(104.5/98.7)=\ln(1.058716)\approx 0.057086\).
2. \(\sigma_{\text{day}}=\frac{|0.057086|}{2\sqrt{\ln 2}}\approx 0.034293\) — **3.4293%** за день.
3. **Шкала 0–100:** \(0.034293\cdot 100\approx 3.4293\Rightarrow 3.43\) (при округлении до 2 знаков).
4. Аннуализация (если нужна вне шкалы): \(0.034293\times\sqrt{252}\approx 0.54439\) — **54.439% годовых**.

### Rogers–Satchell
1. \(\ln(H/C)=\ln(104.5/103.2)\approx 0.012493\).
2. \(\ln(H/O)=\ln(104.5/100.0)\approx 0.044030\).
3. \(\ln(L/C)=\ln(98.7/103.2)\approx -0.044965\).
4. \(\ln(L/O)=\ln(98.7/100.0)\approx -0.013070\).
5. Подкоренное выражение: \(0.012493\cdot 0.044030 + (-0.044965)\cdot (-0.013070)\approx 0.001133\).
6. \(\sigma_{\text{day}}=\sqrt{0.001133}\approx 0.033681\) — **3.3681%** за день.
7. **Шкала 0–100:** \(0.033681\cdot 100\approx 3.3681\Rightarrow 3.37\).
8. Аннуализация (если нужна вне шкалы): \(0.033681\times\sqrt{252}\approx 0.534668\) — **53.467% годовых**.

> В числах выше возможны небольшие расхождения из‑за округлений при печати промежуточных шагов.

---

## Минимальные проверки и обработка крайних случаев
- Если `H ≤ 0` или `L ≤ 0` — данные некорректны для логарифмов, отфильтровать.
- Если `H < L` — ошибка в данных (перепутаны экстремумы).
- Для RS, если из‑за численных ошибок подкоренное выражение получилось отрицательным, берите `max(value, 0)`.
- Если в основной сессии нет сделок — расчёт для этой даты не выполнять. (По условию задачи таких дней нет.)
- При выводе по шкале 0–100 обязательно применять отсечение: `min(100, max(0, value))`.

## Готовые формулы для кода

### Parkinson (рекомендуется для простоты)
```go
// sigmaDay = дневная волатильность по Паркинсону
lnHL := math.Log(H / L)
sigmaDay := math.Abs(lnHL) / (2 * math.Sqrt(math.Log(2)))

// шкала 0–100
score := sigmaDay * 100
if score < 0 { score = 0 }
if score > 100 { score = 100 } // защитное отсечение

// при необходимости округлить до целого 0–100:
scoreInt := int(math.Round(score))

// отдельная аннуализация (если нужна не для шкалы 0–100)
sigmaAnn := sigmaDay * math.Sqrt(252)
```

### Rogers–Satchell (альтернатива, устойчивее к тренду)
```go
v := math.Log(H/C)*math.Log(H/O) + math.Log(L/C)*math.Log(L/O)
if v < 0 { v = 0 } // защита от численных ошибок
sigmaDayRS := math.Sqrt(v)

// шкала 0–100
score := sigmaDayRS * 100
if score < 0 { score = 0 }
if score > 100 { score = 100 }
scoreInt := int(math.Round(score))

// отдельная аннуализация (если нужна не для шкалы 0–100)
sigmaAnnRS := sigmaDayRS * math.Sqrt(252)
```

## Рекомендации по интеграции
- Делайте слой получения данных, который гарантирует: **тикер на TQBR**, **только основная сессия**, **сессия за нужную дату**, **только сделки**.
- Не привязывайтесь к фиксированным часам основной сессии. Используйте биржевые метки/флаги сессии для конкретной даты.
- Округляйте финальные значения на выводе, например до 2 знаков для шкалы 0–100 и до 4–6 — для внутренней \(\sigma_{\text{day}}\) в долях.
