# «Плоский» тренд‑фильтр для тикеров TQBR (MOEX)
Версия: 1.0 • Формат: спецификация расчёта и контрольные примеры • Язык: ru-RU

## Назначение
Оценка отсутствия тренда внутри основной сессии для конкретного тикера TQBR на дату *D*.
Шкала результата: `0…100`, где `0` — сильный тренд, `100` — тренда нет (флэт вокруг VWAP).  
Баланс: простота расчётов > абсолютная точность.

---

## Входные данные
- **Дата**: календарная дата *D* в зоне `Europe/Moscow` (MSK).
- **Тикер**: ц.б. с `boardid = TQBR` (акции и ДР на MOEX).
- **Таймфрейм**: минутные бары основной сессии.
- **Поля бара**: `time, O, H, L, C, V`. Объём `V` в штуках; цены в шагах цены (или рублях) — единицы не важны, но должны быть одни и те же для всех баров.

### Окно данных основной сессии
Использовать только бары, попадающие во временной интервал **\[10:00:00, 18:45:00)** MSK даты *D*.  
Аукционы и внеосновные периоды исключаются. Если в источнике помечены спец‑состояния (auctions, halts) — отфильтровать.

### Предобработка
- Удалить дубликаты меток времени. Если дубли есть — оставить последний по времени прихода.
- Пропуски минут **не заполнять** синтетическими барами. Расчёты ведутся по имеющимся барам.
- Отрицательные или NaN‑значения отклонять; такие бары игнорировать.
- Защита от нулевого диапазона: `ε = 1e-9` (см. ниже).

---

## Обозначения
Пусть отфильтровано `N` минутных баров, пронумеруем их `t = 1..N` по возрастанию времени.
- `H_t, L_t, C_t, V_t` — High, Low, Close, Volume.
- **Типичная цена**:  
  \[1] `P_typ(t) = (H_t + L_t + C_t) / 3`.
- **Кумулятивный VWAP** на момент `t`:
  \[2] `VWAP_t = ( Σ_{i=1..t} V_i * P_typ(i) ) / max( Σ_{i=1..t} V_i , ε )`.
  Если `V_i = 0`, вклад в числитель нулевой; знаменатель не увеличивается.
- **Сессионный диапазон**:  
  \[3] `R = max_t(H_t) - min_t(L_t)`. Затем `R ← max(R, ε)`.
- **Длительность в минутах**: `T_min = N` для минутных баров.

Функция знака:  
`sgn(x) = 1, если x>0; 0, если x=0; -1, если x<0`.

---

## Компоненты скоры

### 1) Пересечения с VWAP
Считаем число смен знака отклонения цены закрытия от VWAP:
\[4] `K = Σ_{t=2..N} 1( sgn(C_t - VWAP_t) ≠ sgn(C_{t-1} - VWAP_{t-1}) )`  
Чтобы нули не давали ложных пересечений, применяем правило: если `sgn` на шаге `t` равен `0`, используем ближайший по времени предыдущий ненулевой знак для сравнения; если такого нет — переход пропускаем.

Нормировка по «ожидаемому» числу пересечений во флэте:
\[5] `K_ref = floor( T_min / 45 )`, минимум `1`.

Скор компонента:
\[6] `S_cross = 100 * min( 1 , K / max(1, K_ref) )`.

Интуиция: во флэте частые возвраты вокруг VWAP → `K` велик → `S_cross` близок к 100.

---

### 2) Доля времени «у VWAP»
Полоса вокруг VWAP шириной `b * R` (по умолчанию `b = 0.10`).  
\[7] `F = (1/N) * Σ_{t=1..N} 1( |C_t - VWAP_t| ≤ b * R )`  
\[8] `S_band = 100 * F`.

Интуиция: чем дольше цена держится в полосе вокруг VWAP, тем более «плоский» рынок.

---

### 3) Сессионный дрейф
Относительный сдвиг закрытия сессии к первому закрытию:
\[9] `D = | C_N - C_1 | / R`  
\[10] `S_drift = 100 * ( 1 - min(1, D) )`.

Интуиция: большой направленный дрейф снижает «плоскость».

---

## Итоговый скор
Взвешенная агрегация и обрезка диапазона:
\[11] `S = clip_{[0,100]}( 0.4*S_cross + 0.4*S_band + 0.2*S_drift )`.

Значение `S` — ответ фильтра: `0` сильный тренд, `100` — флэт.

### Рекомендуемые пороги применения
- `S ≥ 65` — сценарии «возврат к VWAP/средней» допустимы.
- `40 ≤ S < 65` — смешанный режим, снижать размер позиции.
- `S < 40` — тренд; стратегии mean‑reversion нежелательны.

---

## Параметры и тюнинг
- Базовые веса: `0.4 / 0.4 / 0.2`.
- Ширина полосы `b`: 0.10 для ликвидных бумаг. Диапазон 0.08–0.15 по волатильности.
- Таймфрейм 5‑минут: формулы те же, `T_min = 5*N`. `K_ref = floor(T_min/45)` без изменений.
- Робастность: при «пилении» около одной цены `R` становится мал. Защита через `ε` уже учтена.
- Не требуется калибровка под тик‑размер; все метрики относительные к `R` или счетные.

---

## Алгоритм (пошагово)
1. Отфильтровать бары по дате *D* и окну \[10:00, 18:45) MSK. Исключить аукционы.
2. Посчитать `P_typ(t)`, кумулятивно `VWAP_t` по \[2].
3. Найти сессионный `R` по \[3].
4. Подсчитать `K` по \[4] с правилом обработки нулей.
5. `K_ref` по \[5]; `S_cross` по \[6].
6. `S_band` по \[7–8] с `b=0.10` (если не задано иное).
7. `S_drift` по \[9–10].
8. Итог `S` по \[11].

Сложность: `O(N)` по времени, `O(N)` памяти при хранении ряда `VWAP_t`, либо `O(1)` если считать на лету.

---

## Детали реализации
- **Часовой пояс**: вся логика в `Europe/Moscow`. Пересечения дат и переходы на летнее/зимнее время для MSK не актуальны, но используйте явную зону.
- **Аукционы**: если источник отдает флаги стадий, исключать записи с признаками аукционов. Иначе — фильтровать по окну времени и, при необходимости, отбросить нерегулярные метки.
- **Нулевые объёмы**: допустимы; VWAP стабилен, вклад нулевой.
- **Выбросы**: не обрабатываются специально. При желании можно обрезать экстремальные `H/L` по перцентилям перед расчетом `R` — не рекомендуется для базовой версии.
- **Численная стабильность**: для сумм VWAP использовать `float64`.
- **Краевые случаи**:
  - Если `N < 3`, вернуть `S = 0` (недостаточно данных).
  - Если все `H=L=C` в сессии, `R=ε`, формулы вернут `S≈100` (рынок статичен).

## Контрольный микропример (искусственные числа)
Пять минут, `b=0.10`.  
Данные (H,L,C,V):  
t1: (101,  99, 100, 2000)  
t2: (101.5,99.5,100, 1800)  
t3: (102,  99.8,101, 2200)  
t4: (101.2,99.7,100, 1600)  
t5: (101.0,99.6,100, 1500)

1) `P_typ`: t1=100, t2=100.33…, t3=100.93…, t4≈100.3, t5≈100.2.  
2) Кумулятивный VWAP:  
- t1: VWAP₁ = 100  
- t2: VWAP₂ = (2000*100 + 1800*100.33…)/(3800) ≈ 100.17  
- t3: VWAP₃ ≈ 100.49  
- t4: VWAP₄ ≈ 100.40  
- t5: VWAP₅ ≈ 100.33
3) Диапазон `R = max(H) - min(L) = 102 - 99 = 3`.
4) Пересечения: sgn(C - VWAP) по минутам:  
`0, -1, +1, -1, -1` → изменения между t2↔t3 и t3↔t4 → `K=2`.  
`T_min=5`, `K_ref=floor(5/45)=0 → берем 1`.  
`S_cross = 100 * min(1, 2/1) = 100`.
5) Полоса `b*R = 0.3`.  
Все закрытия в пределах `±0.3` от VWAP → `F=1.0` → `S_band=100`.
6) Дрейф: `D = |C5 - C1| / R = |100 - 100|/3 = 0` → `S_drift=100`.
7) Итог: `S = clip( 0.4*100 + 0.4*100 + 0.2*100 ) = 100` → идеальный флэт.

## Скелет реализации на Go
```go
package flatfilter

import "math"

type Bar struct {
	TimeUnix int64   // сек. с начала эпохи в MSK; или храните tz отдельно
	O, H, L, C float64
	V         float64
}

func sign(x float64) int {
	if x > 0 { return 1 }
	if x < 0 { return -1 }
	return 0
}

func FlatTrendScore(bars []Bar, band float64) float64 {
	const eps = 1e-9
	n := len(bars)
	if n < 3 { return 0.0 }

	// 1) Диапазон и VWAP
	var hi, lo float64 = -math.MaxFloat64, math.MaxFloat64
	var num, den float64
	vwaps := make([]float64, n)
	sgn := make([]int, n)

	for i := 0; i < n; i++ {
		b := bars[i]
		if b.H > hi { hi = b.H }
		if b.L < lo { lo = b.L }
		ptyp := (b.H + b.L + b.C) / 3.0
		num += b.V * ptyp
		den += b.V
		if den < eps { vwaps[i] = b.C } else { vwaps[i] = num / den }
		sgn[i] = sign(b.C - vwaps[i])
	}
	R := hi - lo
	if R < eps { R = eps }

	// 2) Пересечения (с учётом нулевых знаков)
	K := 0
	prev := sgn[0]
	for i := 1; i < n; i++ {
		cur := sgn[i]
		if cur == 0 {
			continue // отложим на следующий шаг
		}
		if prev == 0 {
			// найти ближайший предыдущий ненулевой
			j := i-1
			for ; j >= 0 && sgn[j] == 0; j-- {}
			if j >= 0 { prev = sgn[j] } else { prev = cur }
		}
		if cur != prev { K++ }
		prev = cur
	}
	Tmin := float64(n) // минутные бары
	Kref := int(math.Floor(Tmin / 45.0))
	if Kref < 1 { Kref = 1 }
	Scross := 100.0 * math.Min(1.0, float64(K)/float64(Kref))

	// 3) Доля времени в полосе
	if band == 0 { band = 0.10 }
	bw := band * R
	count := 0
	for i := 0; i < n; i++ {
		if math.Abs(bars[i].C - vwaps[i]) <= bw { count++ }
	}
	Sband := 100.0 * float64(count) / float64(n)

	// 4) Дрейф
	D := math.Abs(bars[n-1].C - bars[0].C) / R
	Sdrift := 100.0 * (1.0 - math.Min(1.0, D))

	// 5) Итог
	S := 0.4*Scross + 0.4*Sband + 0.2*Sdrift
	if S < 0 { S = 0 }
	if S > 100 { S = 100 }
	return S
}
```

## Проверочный чек‑лист для интеграции
- Вход: бары только основной сессии даты *D* и борда TQBR.
- Единицы цены и объёма согласованы по всем барам.
- Параметр `band` либо 0.10, либо задаётся явно.
- Типы `float64` для всех расчётов.
- Локаль времени — MSK для резки сессии.
- Логи или метрики: `N`, `R`, `K`, `K_ref`, `F`, `D`, `S_*`, финальный `S`.

---

## Выход
Единственное число `S` в диапазоне `0…100`.
Рекомендуется логировать и компоненты `S_cross`, `S_band`, `S_drift` для диагностики.

---

## Лицензия и оговорки
Спецификация предоставлена «как есть». Подходит для автоматизации интрадей фильтра mean‑reversion к VWAP без претензии на универсальность.
