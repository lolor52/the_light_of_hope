# Ликвидность — расчёт за Основную сессию MOEX

> Все расчёты выполняются **только по Основной торговой сессии MOEX**. Утренняя и вечерняя сессии исключены. **Аукционы открытия и закрытия также исключены.**

## Входные параметры
```
Ticker = <код тикера, один>
Date   = <YYYY-MM-DD> (MSK)
S      = множество тикеров для кросс-нормировки на Date
```
> Если нужна оценка только для одного тикера, множество `S` всё равно задаётся, но нормировка защищена `ε`.

## Источники данных (MOEX ISS)
- **marketdata**: `BID`, `OFFER`
- **orderbook**: `bids[i].price`, `bids[i].quantity`, `asks[i].price`, `asks[i].quantity`
- **candles 1m**: `begin`, `value` (денежный оборот за минуту), `volume`
- **securities**: `LOTSIZE` (размер лота для тикера, требуется для корректных единиц глубины)
- **расписание/борды**: границы Основной сессии и интервалы аукционов для соответствующего `engine/market/board`

## Окно расчёта
Пусть `T_open` и `T_close` — границы Основной сессии для выбранного `board` **без учёта аукционов**. Тогда:
- Для свечей 1m: используем только бары с `begin ∈ [T_open, T_close)`.
- Для marketdata и orderbook: используем записи/снапшоты с `ts ∈ [T_open, T_close)` и **исключаем интервалы аукционов** согласно расписанию борда.

## Метрики (только уточнения к текущей версии)
### 1) Глубина стакана TOP‑5: дискретизация и единицы
**Дискретизация во времени.** Для каждой календарной минуты `m = [t_m, t_{m+1})` берём все снапшоты стакана в этой минуте и считаем минутное значение глубины как TWAP по времени между снапшотами.
Если в минуте нет снапшотов, минута пропускается.
Также **отбрасываем минуты**, где отсутствует хотя бы один из лучших сторон (`BID` или `OFFER`) или их объём равен нулю.

**Единицы.** Считаем два варианта глубины TOP‑5 на минуте `m`:
```
Depth_TOP5_QTY[m]   = Σ_{k=1..5} (bids_k.quantity + asks_k.quantity) * LOTSIZE
Depth_TOP5_MONEY[m] = Σ_{k=1..5} (bids_k.price*bids_k.quantity
                                  + asks_k.price*asks_k.quantity) * LOTSIZE
```
Сессионные агрегаты (средние по минутам окна):
```
Depth_TOP5_QTY_sess   = mean_m Depth_TOP5_QTY[m]
Depth_TOP5_MONEY_sess = mean_m Depth_TOP5_MONEY[m]
```

> Для межтикерной сопоставимости в дальнейшем используем **денежную** глубину `Depth_TOP5_MONEY_sess` в нормировке и скоринге. Количественная `Depth_TOP5_QTY_sess` остаётся как вспомогательная.

### 2) Исключение аукционов
Во всех расчётах по marketdata, orderbook и свечам 1m исключаем интервалы аукционов открытия и закрытия. Это устраняет несопоставимые всплески объёма/глубины.

## Нормировки
Нормировки остаются без изменений, но теперь применяются к `Depth_TOP5_MONEY_sess`:
```
norm_up(x_i)   = (x_i - p10_S) / (p90_S - p10_S + ε),  обрезка в [0,1]
norm_down(x_i) = 1 - norm_up(x_i)
```
где квантили считаются по множеству `S` на дату `Date`.

## Интегральный скор (фрагмент с поправкой на глубину)
Остальная формула скоринга без изменений; заменяем только компоненту глубины:
```
... + 0.25*norm_up(Depth_TOP5_MONEY_sess_i)
    + 0.05*norm_down(Tick%_sess_i) ]
```

## Параметры
```
ε = 1e-9
N уровней стакана для глубины: 5
MainSessionOnly = true
ExcludeAuctions = true
TZ = MSK (UTC+3)
```

## Выход
Одно число `L_i(Ticker, Date) ∈ [0, 100]` — оценка ликвидности.
При необходимости можно возвращать промежуточные сессионные метрики.
