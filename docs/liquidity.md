# Ликвидность — расчёт за Основную сессию MOEX

## Цель
Получить одно число `L(TICKER, DATE) ∈ [0,100]`, оценивающее ликвидность инструмента на **Основной торговой сессии** в указанную дату.

## Вход
```
TICKER  = <SECID>
BOARD   = <BOARDID>
DATE    = <YYYY-MM-DD> (MSK)
S       = множество тикеров для кросс-нормировки на DATE (тот же BOARD и класс, включает TICKER)
```
> Все вычисления в часовом поясе MSK (UTC+3).

## Используемые ресурсы MOEX ISS
- `.../engines/<engine>/markets/<market>/boards/<board>/securities/<SECID>.json?iss.only=securities` — справочник (в т.ч. `LOTSIZE`, `MINSTEP`, валюта расчёта).
- `.../engines/<engine>/markets/<market>/boards/<board>/securities/<SECID>/candles.json?from=DATE&to=DATE&interval=1` — минутные свечи основной даты.
- `.../engines/<engine>/markets/<market>/boards/<board>/securities/<SECID>/trades.json?date=DATE` — тиковые сделки за дату.
- `.../engines/<engine>/markets/<market>/boards/<board>/sessions.json?date=DATE` или `.../boards/<board>/schedule` — расписание сессий для отбора Основной.
- `.../engines/<engine>/markets/<market>/boards/<board>/securities.json?iss.only=securities` — универсум бумаг для множества `S` (фильтры: `is_traded`, класс бумаги, `BOARD`).

## Границы Основной сессии и минутный грид
1) По `sessions` определить точные **начала и окончания Основной сессии** для `BOARD` и `DATE`. Явно исключить все аукционы и продления.
2) Построить **полный минутный грид** `M = {t_i}` с шагом 1 минута на интервале Основной сессии.
3) Запросить `candles?interval=1` за `DATE`, **реиндексировать** по `M`:
   - если минута в ответе отсутствует, задать `volume_m=0`, `value_m=0`;
   - цены заполнить по правилу **forward-fill** `close` от последнего известного значения.
   - если до первой сделки нет цен, базовую цену взять в порядке приоритета:
     (a) цена открытия Основной сессии из свечи, если дана; (b) цена первой сделки на `DATE` из `trades`; (c) **предыдущий дневной close** из `history` этой бумаги на MOEX ISS.
   - для пустых минут задать `open_m = high_m = low_m = close_m` равными заполненному `close_m`.
4) Обозначить `M_act = { m ∈ M | volume_m > 0 }` — активные минуты.

## Поля и единицы
- `volume_m` трактуется как количество **лотов**. Денежный оборот рассчитывается в валюте инструмента.
- При наличии `value_m` в свечах использовать его. Если поля нет: `value_m := volume_m * LOTSIZE * price_proxy_m`, где `price_proxy_m := (high_m + low_m + 2*close_m)/4`.
- `LOTSIZE`, `MINSTEP`, валюта — из `securities` по `TICKER` на `BOARD`.

## Минутные величины
- Лог-ретёрн: `r_m = | ln( close_m / close_{m-1} ) |` для последовательных минут из `M`. Пары без цены пропускаются.
- Изменение цены: `Δp_m = close_m - close_{m-1}`.
- Базовая цена: `price_ref = median( { close_m | m ∈ M_act } )`.

## Сессионные метрики
1) Суммарный оборот: `V_total = Σ_{m∈M} value_m`.
2) Медианный минутный оборот по активным минутам: `V_med = median( { value_m | m ∈ M_act } )` (если `M_act` пусто — NA).
3) Доля активных минут: `A = |M_act| / |M|`.
4) **Amihud (1m):** `ILLIQ = mean( { r_m / (value_m + ε) | m ∈ M_act и r_m определён } )`.
5) **Roll spread proxy (тик-уровень, ISS trades):**
   - Взять последовательность цен сделок `P_k` из `trades` за `DATE` (по `BOARD`, без аукционов если поле доступно; иначе фильтровать по времени Основной сессии).
   - Рассчитать `Δp_k = P_k - P_{k-1}` для соседних тиков в пределах Основной сессии.
   - `R = 2 * sqrt( max(0, - cov(Δp_k, Δp_{k-1}) ) )` при наличии ≥3 тиков; если ковариация ≥0 или тиков мало — метрика NA и будет импутирована.
   - **Fallback**: если `trades` недоступны или <3 тиков, допускается расчёт Roll на минутных `Δp_m` с пометкой `roll_from_candles=true`.
6) Крупность тика: `TickPct = 10000 * MINSTEP / price_ref`.
7) **Прокси глубины (устойчивый):**
   - Защита от малых `r_m`: определить `τ = quantile( { r_m | m ∈ M_act }, 5% )`. Винзорировать `r_m^* = max(r_m, τ)`.
   - `DepthProxy = median( { value_m / r_m^* | m ∈ M_act и r_m определён } )`.
   - Альтернатива: винзорировать отношение на уровне 1% и 99% квантилей по минутам.

Константы: `ε = 1e-12` (числовая защита).

## Нормировка по множеству S
Множество `S` формируется **только из бумаг того же класса и того же `BOARD`**, торгуемых на `DATE` (`is_traded=true` в `securities`). Для каждой метрики `X` (из {`V_total`, `V_med`, `DepthProxy`, `A`, `ILLIQ`, `R`, `TickPct`}):

- Квантили: `P5_X = quantile_S(X, 5%)`, `P95_X = quantile_S(X, 95%)` по значениям на `DATE`.
- Нормировки:
  - Вверх-лучше: `norm_up_X(x) = clip( (x - P5_X) / (P95_X - P5_X), 0, 1 )`.
  - Вниз-лучше: `norm_down_X(x) = clip( (P95_X - x) / (P95_X - P5_X), 0, 1 )`.
  - **Спец-случай:** если `P95_X == P5_X`, вернуть `0.5` для всех значений `x` и зафиксировать флаг `flat_quantiles_X=true`.
- Требование к размеру: если `|S| < 10`, нормировки допускаются, но отчёт помечается `small_peer_set=true`.

## Итоговый скор
Весовая схема сохраняется как в прежней версии (при необходимости замените на вашу конфигурацию весов):
```
L_raw = w_Vtotal * norm_up_V_total(V_total)
      + w_Vmed   * norm_up_V_med(V_med)
      + w_Depth  * norm_up_DepthProxy(DepthProxy)
      + w_A      * norm_up_A(A)
      + w_ILLIQ  * norm_down_ILLIQ(ILLIQ)
      + w_Roll   * norm_down_R(R)
      + w_Tick   * norm_down_TickPct(TickPct)

L = round( 100 * clip(L_raw, 0, 1), 2 )
```
`L ∈ [0, 100]`.

## Правила пропусков и отчётность
- Если после отбора по Основной сессии `|M| < 60` минут, вернуть `L = NaN`, статус `insufficient_session`.
- Метрика отсутствует ⇒ использовать **среднее по нормировке** `0.5` для этой метрики и флаг `imputed_X=true`.
- Для `R`: при отсутствии пригодных тиков использовать fallback на минутных данных с флагом `roll_from_candles=true`.
- Все импутации и флаги фиксировать в отчёте: размеры выборок, `|M_act|/|M|`, наличие `flat_quantiles`, `small_peer_set`, `roll_from_candles`, доля винзорированных минут в `DepthProxy`.

## Проверка соответствия ограничениям
- Основным источником данных является MOEX ISS (`candles`, `trades`, `securities`, `sessions/schedule`). Данные стакана и сторонние поставщики не используются.
- Пустые минуты корректно представлены в гриде `M` и **не завышают** метрики.
- Нормировки устойчивы при `P5==P95`.
- Прокси глубины устойчива к малым ретёрнам.
- Roll рассчитывается на тиках ISS, что минимизирует ложные нули при редких сделках.

## Параметры
```
ε = 1e-12
TZ = MSK (UTC+3)
QuantilesPeer = (5%, 95%)
τ (порог для r_m) = 5%-квантиль по активным минутам (либо фикс. минимум по бизнес-правилу)
MinMinutesForSession = 60
MinTradesForRoll = 3
```

## Выход
Одно число `L(TICKER, DATE)` и служебный отчёт качества.
