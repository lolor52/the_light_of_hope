
# Метрика ликвидности для акций **TQBR** за основную сессию в конкретную дату

Версия: 2025-10-22

## 1) Определение метрики

Базовая метрика (простая и быстрая к расчёту):

$$
L_{{\text{{simple}}}}=\frac{{\text{{TV}}}}{{\dfrac{{H-L}}{{\text{{VWAP}}}}+\varepsilon}}
$$

Где:
- **TV** = \(\sum_i p_i q_i\) — оборот за основную сессию, руб.
- **VWAP** = \(\dfrac{{\sum_i p_i q_i}}{{\sum_i q_i}}\).
- **H, L** — максимум и минимум цен сделки за основную сессию.
- \(\varepsilon=10^{{-6}}\) — числовая защита от нуля.
- Единицы: рубли. Для сравнимости между тикерами используйте \(\operatorname{{log1p}}(L_{{\text{{simple}}}})\).

Интерпретация: большой оборот при узком внутридневном диапазоне ⇒ высокая ликвидность.


## 2) Область применения

- Тикер торгуется на **Московской бирже**, **boardid = TQBR**.
- Дата торгов указана и в эту дату были сделки.
- Приоритет — баланс точности и простоты.


## 3) Окно данных (только основная сессия)

Фильтрация по времени в **MSK (UTC+3)**: включительно с `10:00:00` и до `18:45:00` **не включая** 18:45:00.

Псевдокод фильтра:
```
trade.boardid == "TQBR" &&
trade.time >= "10:00:00" &&
trade.time <  "18:45:00" &&
trade.date == <заданная дата в MSK>
```

Виды данных:
- **Тиковые сделки**: время, цена `p_i`, количество `q_i`.
- Допустимо считать из аггрегированных минуток, если внутри минуты есть VWAP и оборот (см. §7).


## 4) Алгоритм расчёта L_simple

1. Отфильтровать сделки по §3.
2. Посчитать:
   - \(\text{{TV}}=\sum_i p_i q_i\), руб.
   - \(Q=\sum_i q_i\).
   - \(\text{{VWAP}}=\text{{TV}}/Q\).
   - \(H=\max_i p_i\), \(L=\min_i p_i\).
3. Диапазон в относительных единицах: \(R=(H-L)/\text{{VWAP}}\).
4. Числовая защита: \(\varepsilon=10^{{-6}}\).
5. Итог: \(L_{{\text{{simple}}}}=\text{{TV}}/(R+\varepsilon)\).
6. Нормировка для сравнений между именами: \(L_{{\text{{simple,log}}}}=\operatorname{{log1p}}(L_{{\text{{simple}}}})\).


## 5) Нормировка к шкале **0–100**

Итоговый балл \(S\in[0,100]\). Больше — ликвиднее.

### Вариант A (предпочтительный): кросс-секционный перцентиль за день
1. На указанную дату посчитать \(X=\operatorname{{log1p}}(L_{{\text{{simple}}}})\) для всех тикеров **TQBR**.
2. Пусть \(A=P10(X)\), \(B=P90(X)\) по всему списку тикеров за этот день.
3. Балл:
$$
S = 100\cdot \operatorname{{clip}}\left(\frac{X-A}{B-A},\ 0,\ 1\right).
$$
Где `clip(z,0,1)=min(max(z,0),1)`.

Свойства: монотонность, устойчивость к выбросам, автоматическая «растяжка» под рыночный день.

### Вариант B: исторические перцентели по тикеру
Если нет кросс-секции за день, используйте по тикеру последние \(N=60\) основных сессий: \(A=P10(X), B=P90(X)\) по истории. Формула для \(S\) та же.

### Вариант C: параметрическая логистическая аппроксимация
Если нет перцентилей, используйте насыщаемую функцию:
$$
S = 100\cdot \left(1-e^{-X/\mu}\right),\quad X=\operatorname{{log1p}}(L_{{\text{{simple}}}}).
$$
Параметр масштаба:
- \(\mu=\operatorname{{median}}(X)\) по последним \(N=60\) сессиям тикера, либо
- константа \(\mu=10\) как дефолт.

Округляйте результат до целого: \(\operatorname{{round}}(S)\).

**Пример (логистический, \(\mu=10\)) для синтетических данных ниже:** `S = 70.94` из 100.


## 6) Практические уточнения и проверки качества

- **Мин. насыщенность**: если число сделок < 30, пометить результат как пониженной надёжности.
- **Нулевой диапазон**: если \(H=L\) или все сделки по одной цене, то \(R=0\) и \(L_{{\text{{simple}}}}\approx \text{{TV}}/\varepsilon\). Это корректно трактуется как «очень ликвидно» при большом обороте; в логистике это даёт \(S\) близкий к 100.
- **Выбросы**: при одиночных ошибочных ценах допустимо винсоризовать \(H\) и \(L\) по 0.1-й и 99.9-й перцентилям внутри дня.
- **Валюта**: все расчёты в рублях.
- **Корректировки**: сплиты и дивиденды внутри дня не влияют на внутридневной диапазон, поэтому корректировки не требуются.
- **Сортировка**: использовать фактический порядок по времени сделки.


## 7) Рабочий пример (синтетические сделки)

Данные:
```
(p, q): (100.00, 10), (101.00, 5), (99.50, 20)
```
Расчёты:
- TV = 3495.00 руб
- Q = 35
- VWAP = 99.857143
- H = 101.00, L = 99.50
- R = (H-L)/VWAP = 0.015021
- \(L_{{\text{{simple}}}}\) = TV / (R + \varepsilon) = 232 651.655 руб
- \(X=\operatorname{{log1p}}(L_{{\text{{simple}}}})\) = 12.357302
- Пример логистического балла при \(\mu=10\): **S = 70.94** / 100


## 8) Вариант точнее: инвертированный Amihud по минутам

Определения по минуте \(m\):
- \(P_m\) — минутный VWAP.
- \(V_m\) — минутный оборот, руб.
- \(r_m = |\ln P_m - \ln P_{m-1}|\), минуты без сделок пропустить.
- Нижний порог оборота: \(V_\text{{floor}}=1000\) руб.

Промежуточная метрика неликвидности (Amihud):
$$
\text{{ILLIQ}}=\operatorname{{median}}_m \left( \frac{{r_m}}{{\max(V_m, V_\text{{floor}})}} \right)
$$

Итоговая «ликвидность» для интерпретации «чем больше — тем лучше»:
$$
L_{{\text{{amihud}}}}=\frac{1}{\text{{ILLIQ}}},\quad
L_{{\text{{amihud,log}}}}=\operatorname{{log1p}}(L_{{\text{{amihud}}}})
$$

Короткий числовой пример (синтетический):
- \(P = [100.0, 100.8, 100.2]\)
- \(V = [200000, 150000, 300000]\)
- \(r = [|\ln 100.8 - \ln 100.0|, |\ln 100.2 - \ln 100.8|]\)
- Термы Amihud: \([r_1/V_1,\ r_2/V_2]\) = [5.312113099451e-08, 1.990055662168e-08]
- \(\text{{ILLIQ}}\) = 5.312113099451e-08
- \(L_{{\text{{amihud}}}}\) = 18824900.397986
- \(\operatorname{{log1p}}(L_{{\text{{amihud}}}})\) = 16.750691


## 9) Реализация в Go: минимальные шаблоны

```go
func Clip01(x float64) float64 {
    if x < 0 { return 0 }
    if x > 1 { return 1 }
    return x
}

func ScoreFromPercentiles(x, a, b float64) float64 {
    if b <= a { return 0 } // защита
    z := (x - a) / (b - a)
    return 100 * Clip01(z)
}

func ScoreLogistic(x, mu float64) float64 {
    if mu <= 0 { mu = 10 } // дефолт
    return 100 * (1 - math.Exp(-x/mu))
}
```

Встраивание с расчётом \(L_{{\text{{simple}}}}\):
```go
// после вычисления L_simple:
x := math.Log1p(L_simple)

// A) кросс-секция дня: a := P10(X_all), b := P90(X_all); score := ScoreFromPercentiles(x, a, b)

// B) история тикера: a := P10(X_hist_60), b := P90(X_hist_60); score := ScoreFromPercentiles(x, a, b)

// C) без перцентилей: score := ScoreLogistic(x, mu) // mu=median(X_hist_60) или 10
scoreInt := math.Round(score) // 0..100
```


## 10) Выходные поля сервиса

- `ticker` (string)
- `date` (YYYY-MM-DD, MSK)
- `score_0_100` (int) — итоговый балл 0–100
- `L_simple` (float64, руб)
- `L_simple_log1p` (float64)
- `trades_count` (int)
- `turnover` (TV, руб, float64)
- `vwap` (float64)
- `high`, `low` (float64)
- `range_rel` (float64)
- Опционально: `L_amihud`, `L_amihud_log1p`
- Диагностика: использованный метод нормировки и параметры (A,B) или `mu`


## 11) Краевые случаи

- Все сделки по одной цене ⇒ диапазон 0 ⇒ \(X\) велик ⇒ балл близок к 100 при методе C.
- Очень редкие сделки ⇒ помечайте низкую надёжность, но метрика и балл считаются.
- Отсутствие тиков в основную сессию в указанную дату (теоретически) ⇒ `ok=false`.
- Только `boardid=TQBR`.
- Нулевые или отрицательные количества исключать на этапе загрузки данных.
