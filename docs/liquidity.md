# Ликвидность — расчёт за Основную сессию MOEX

> Все расчёты выполняются **только по Основной торговой сессии MOEX**. Утренняя и вечерняя сессии исключены. **Аукционы открытия и закрытия также исключены.**

## Входные параметры
```
Ticker = <код тикера, один>
Date   = <YYYY-MM-DD> (MSK)
S      = множество тикеров для кросс-нормировки на Date
```
> Если нужна оценка только для одного тикера, множество `S` всё равно задаётся, но нормировка защищена `ε`.

## Источник данных
- Только **MOEX ISS HTTP API** (`iss.moex.com`).
- Используемые конечные точки: `securities`, `candles` (1m+), `boards/schedule`.
- Потоковые или сторонние источники не используются.
- Стакан не используется; все метрики считаются по свечам 1m и расписанию борда из ISS.


## Окно расчёта
Пусть `T_open` и `T_close` — границы Основной сессии для выбранного `board` **без учёта аукционов**. Тогда:
- Для свечей 1m: используем только бары с `begin ∈ [T_open, T_close)`.

## Метрики (адаптация под ISS без стакана)
### Прокси глубины по свечам 1m
Для каждой календарной минуты `m` внутри окна:
- Цена: `p_m = close_m`.
- Возврат: `r_m = 100 * |ln(p_m / p_{m-1})|` в процентах; минуты без пары цен исключаем.
- Оборот: если в свечах есть `value_m`, берём его; иначе `V_m = volume_m * LOTSIZE * ((high+low+2*close)/4)`.
- Прокси глубины на минуте: `DepthProxy_m = V_m / (r_m + ε)`; минуты с `volume_m=0` исключаем.

Сессионный агрегат:
- `DepthProxy_MONEY_sess = mean_m DepthProxy_m`.

Исключение аукционов: из минутного набора удаляются интервалы аукционов по `boards/schedule`.

## Интегральный скор (фрагмент с поправкой на глубину)
Остальная формула скоринга без изменений; заменяем только компоненту глубины:
```
... + 0.25*norm_up(DepthProxy_MONEY_sess_i)
    + 0.05*norm_down(Tick%_sess_i) ]
```

## Параметры
```
ε = 1e-9
MainSessionOnly = true
ExcludeAuctions = true
TZ = MSK (UTC+3)
```

## Выход
Одно число `L_i(Ticker, Date) ∈ [0, 100]` — оценка ликвидности.
При необходимости можно возвращать промежуточные сессионные метрики.
