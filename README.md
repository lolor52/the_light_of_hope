# invest_intraday

Прототип внутреннего сервиса для подготовки к внутридневной торговле на нескольких рынках.

## Требования

- Go 1.24.5
- PostgreSQL 17.5

## Запуск

```bash
go run ./...
```

## Конфигурация

Основные параметры задаются в `config.json`:

- `auth.alor` — данные для авторизации в АЛОР OpenAPI.
- `selection` — параметры отбора рынков по волатильности.
- `markets` — исторические значения волатильности по рынкам.
- `order_defaults` — шаблон заявки, используемый при старте торгов.
- `session` — официальные границы сессии.
- `vwap_inputs` — данные для расчёта VWAP и value area.

Значения можно адаптировать под конкретную стратегию без перекомпиляции программы.

## Модули

- `main.go` — точка входа, объединяет загрузку конфигурации, авторизацию, выбор рынков, расчёт VWAP и подготовку заявки.
- `internal/auth/alor` — получение access token по refresh token для API АЛОР.
- `internal/market_prelaunch_selection` — расчёт волатильности за последние сессии и выбор рынков для старта.
- `internal/order` — валидация параметров и построение сводного описания заявки.
- `internal/a_submodule/session` — вычисление используемых границ торгового окна относительно официальной сессии.
- `internal/a_submodule/variable` — расчёты VWAP и границ value area по различным наборам данных.
- `internal/a_submodule/vah_gt_vwap_gt_val` — проверка инварианта отношения VAH/VWAP/VAL для результирующего набора.
